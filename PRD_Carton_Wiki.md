# Carton Wiki 模块产品需求文档 (PRD)

## 1. 产品目标 (Product Goals)

### 核心目标
1. **知识赋能** - 为外贸销售人员提供专业、准确的纸箱类型知识库，提升客户沟通的专业度和说服力
2. **快速查询** - 建立结构化的纸箱百科全书，支持快速搜索和对比，提高工作效率
3. **决策辅助** - 通过对比功能和详细参数，帮助销售人员为客户推荐最合适的纸箱类型

### 成功指标
- 用户每周访问Carton Wiki模块至少3次
- 搜索功能使用率 > 60%
- 对比功能使用率 > 30%
- 用户反馈满意度 > 4.5/5

---

## 2. 用户故事 (User Stories)

1. **As a 外贸销售新手**, I want 查看各种纸箱类型的详细信息和图示, so that 我能快速学习纸箱专业知识，在客户面前显得专业

2. **As a 外贸销售人员**, I want 根据纸箱类型名称快速搜索相关信息, so that 在客户询问时能立即找到答案

3. **As a 销售顾问**, I want 对比2-4种纸箱类型的参数差异, so that 我能为客户推荐最适合其需求的纸箱类型

4. **As a 报价人员**, I want 查看每种纸箱类型的成本估算和生产设备要求, so that 我能准确报价和评估交货期

5. **As a 客户经理**, I want 了解每种纸箱的应用场景和优缺点, so that 我能向客户解释为什么推荐某种纸箱类型

---

## 3. 功能需求清单 (Requirements Pool)

### P0 - Must Have (核心功能)

#### 3.1 侧边栏导航
- **REQ-001**: 在现有侧边栏添加"纸箱Wiki"导航入口，位置在"纸箱计算"之后
- **REQ-002**: 点击"纸箱Wiki"进入模块主页，高亮显示当前选中状态
- **REQ-003**: 侧边栏入口显示纸箱图标（使用 MUI Material Icons 的 `description` 或 `menu_book` 图标）

#### 3.2 纸箱类型百科全书
- **REQ-004**: 主页展示所有纸箱类型的卡片网格（Card Grid），至少包含15种常见类型
- **REQ-005**: 每种纸箱类型必须包含以下信息：
  - 标准代码（如 RSC, HSC, FOL 等）
  - 英文名称和标准中文名称
  - 结构描述（text）
  - 应用场景（至少3个典型场景）
  - 优点（至少3点）
  - 缺点（至少2点）
  - 制作成本估算（低/中/高 三档）
  - 所需生产设备（列举主要设备）
  - 适用瓦楞纸板类型（A/B/C/E瓦楞，单层/双层/三层）
  - **展开图**：使用SVG或ASCII art展示纸箱展开后的平面结构
- **REQ-006**: 点击纸箱类型卡片进入详情页面，展示完整信息
- **REQ-007**: 详情页面包含返回按钮，返回主页

#### 3.3 搜索功能
- **REQ-008**: 主页顶部显示搜索框，支持实时搜索（debounce 300ms）
- **REQ-009**: 支持按纸箱类型名称、标准代码、应用场景搜索
- **REQ-010**: 搜索无结果时显示友好提示："未找到相关纸箱类型，请尝试其他关键词"
- **REQ-011**: 搜索结果高亮显示匹配关键词

### P1 - Should Have (重要功能)

#### 3.4 对比功能
- **REQ-012**: 主页和详情页提供"对比"复选框，最多可选择4种纸箱类型
- **REQ-013**: 选择2种及以上类型后，激活"开始对比"按钮
- **REQ-014**: 对比页面使用表格展示以下维度：
  - 标准代码
  - 外摇盖长度（mm，典型尺寸）
  - 顶面/底面层数（单层/双层）
  - 抗压强度（低/中/高）
  - 材料成本（1-5级，5为最高）
  - 组装难度（简单/中等/复杂）
  - 封闭性（差/一般/好/优秀）
  - 市场占比（%）
  - 核心用途（简要描述）
- **REQ-015**: 对比表格支持横向滚动（移动端适配）
- **REQ-016**: 对比页面提供"导出对比表"功能（复制为Markdown格式）

#### 3.5 UI/UX 优化
- **REQ-017**: 遵循苹果设计风格：简洁、现代、圆角卡片、阴影效果
- **REQ-018**: 响应式设计：适配桌面端（>1024px）、平板（768-1024px）、手机（<768px）
- **REQ-019**: 纸箱类型卡片使用悬停效果（scale 1.02, shadow加深）
- **REQ-020**: 页面切换使用平滑过渡动画（0.3s ease）

### P2 - Nice to Have (增强功能)

#### 3.6 高级搜索和过滤
- **REQ-021**: 添加筛选器：按应用场景、成本等级、抗压强度筛选
- **REQ-022**: 支持高级搜索：组合多个条件（如"成本低 AND 抗压强度高"）

#### 3.7 收藏和笔记
- **REQ-023**: 用户可以收藏常用纸箱类型（保存到本地localStorage）
- **REQ-024**: 用户可以为每种纸箱类型添加个人笔记（保存到本地localStorage）

#### 3.8 数据导出
- **REQ-025**: 详情页面提供"导出为PDF"功能
- **REQ-026**: 提供"分享链接"功能（生成该纸箱类型的唯一URL hash）

## 4. 纸箱类型数据库 (Carton Type Database)

### 4.1 RSC (Regular Slotted Carton) - 常规开槽纸箱
- **标准代码**: RSC
- **英文名称**: Regular Slotted Carton
- **中文名称**: 常规开槽纸箱 / 标准开槽箱
- **展开图**:
```
    ┌─────────────────────────────────────┐
    │                                     │
    │  ┌────┐       ┌────┐       ┌────┐ │
    │  │    │       │    │       │    │ │
    ├──┘    └───────┘    └───────┘    └─┤
    │                                     │
    │  [     箱底面（单层）]              │
    │                                     │
    ├──┐    ┌───────┐    ┌───────┐    ┌─┤
    │  │    │       │    │       │    │ │
    │  └────┘       └────┘       └────┘ │
    │                                     │
    └─────────────────────────────────────┘
      ↑外摇盖        内摇盖↓
```
- **结构特点**: 
  - 四个摇盖长度相等或成比例
  - 内外摇盖交错搭接
  - 最通用的纸箱结构
- **应用场景**: 
  1. 快递物流包装（最常用）
  2. 食品饮料包装
  3. 电子产品包装
  4. 日用百货运输
- **优点**:
  1. 成本最低，适合大批量生产
  2. 结构简单，易于自动化组装
  3. 适用面广，90%的纸箱都是RSC
- **缺点**:
  1. 抗压强度一般（摇盖搭接处易变形）
  2. 密封性较差（需胶带封口）
  3. 不适合重型或易碎品
- **制作成本估算**: 低（1/5）
- **所需生产设备**: 
  - 瓦楞纸板生产线
  - 印刷开槽机（Flexo Folder Gluer）
  - 自动打包机
- **适用瓦楞纸板**: 
  - 类型：A瓦、B瓦、C瓦、E瓦、BC瓦、AB瓦
  - 层数：单层（3层）、双层（5层）、三层（7层）
- **典型尺寸**: 外摇盖长度 = 箱宽/2
- **抗压强度**: 中
- **组装难度**: 简单
- **封闭性**: 一般（需封箱胶带）
- **市场占比**: ~60%

### 4.2 HSC (Half Slotted Carton) - 半开槽纸箱
- **标准代码**: HSC
- **英文名称**: Half Slotted Carton
- **中文名称**: 半开槽纸箱
- **展开图**:
```
    ┌─────────────────────────────────────┐
    │     （无顶盖，只有三个摇盖）         │
    │  ┌────┐       ┌────┐              │
    │  │    │       │    │              │
    ├──┘    └───────┘    └──────────────┤
    │                                     │
    │  [     箱底面（单层）]              │
    │                                     │
    ├──┐    ┌───────┐    ┌──────────────┤
    │  │    │       │    │              │
    │  └────┘       └────┘              │
    │                                     │
    └─────────────────────────────────────┘
      ↑只有底摇盖     （顶部开口）
```
- **结构特点**: 
  - 只有底部摇盖，顶部完全开口
  - 通常用于需要频繁取放物品的场景
- **应用场景**: 
  1. 零售展示箱（Display Box）
  2. 仓库储物箱
  3. 水果蔬菜采摘箱
  4. 自动化生产线进料箱
- **优点**:
  1. 顶部开口，便于取放物品
  2. 可折叠平放，节省储存空间
  3. 成本较低
- **缺点**:
  1. 无顶盖保护，防尘防水差
  2. 堆叠时需注意稳定性
  3. 不适合长途运输
- **制作成本估算**: 低（1/5）
- **所需生产设备**: 印刷开槽机、模切机（如需特殊形状）
- **适用瓦楞纸板**: A瓦、B瓦、C瓦、BC瓦；单层、双层
- **抗压强度**: 中低
- **组装难度**: 简单
- **封闭性**: 差
- **市场占比**: ~5%

### 4.3 FOL (Full Overlap Carton) - 全叠盖纸箱
- **标准代码**: FOL
- **英文名称**: Full Overlap Carton
- **中文名称**: 全叠盖纸箱 / 全重叠纸箱
- **展开图**:
```
    ┌─────────────────────────────────────┐
    │                                     │
    │  ┌─────────┐   ┌─────────┐        │
    │  │         │   │         │        │
    ├──┘         └───┘         └────────┤
    │   摇盖完全重叠（100%覆盖）          │
    │                                     │
    │  [     箱底面（双层）]              │
    │                                     │
    ├──┐         ┌───┐         ┌────────┤
    │  │         │   │         │        │
    │  └─────────┘   └─────────┘        │
    │                                     │
    └─────────────────────────────────────┘
      ↑外摇盖长度 = 箱宽（完全重叠）
```
- **结构特点**: 
  - 外摇盖长度等于箱宽，完全重叠
  - 箱底面为双层纸板，强度更高
- **应用场景**: 
  1. 重型机械零部件包装
  2. 玻璃陶瓷等易碎品
  3. 长途运输出口包装
  4. 高价值电子产品
- **优点**:
  1. 抗压强度高（双层底面）
  2. 密封性好（摇盖完全重叠）
  3. 耐磨损，适合恶劣运输环境
- **缺点**:
  1. 材料用量多，成本增加20-30%
  2. 重量增加
  3. 组装稍复杂（需折叠双层摇盖）
- **制作成本估算**: 中高（3/5）
- **所需生产设备**: 印刷开槽机、自动折叠粘合机
- **适用瓦楞纸板**: B瓦、C瓦、BC瓦、AB瓦；双层（5层）、三层（7层）
- **抗压强度**: 高
- **组装难度**: 中等
- **封闭性**: 好
- **市场占比**: ~8%

### 4.4 Telescope Box - 套盖纸箱
- **标准代码**: TEL
- **英文名称**: Telescope Box
- **中文名称**: 套盖纸箱 / 天盖地盒
- **展开图**:
```
    顶盖（Telescope Top）
    ┌─────────────────────┐
    │                     │
    │  ┌─────────────┐   │
    │  │             │   │
    │  │   （内嵌）  │   │
    │  │             │   │
    │  └─────────────┘   │
    │                     │
    └─────────────────────┘
            ↓套入
    底盒（Telescope Base）
    ┌─────────────────────┐
    │                     │
    │  ┌─────────────┐   │
    │  │             │   │
    │  │             │   │
    │  │             │   │
    │  └─────────────┘   │
    │                     │
    └─────────────────────┘
```
- **结构特点**: 
  - 分离式顶盖和底盒
  - 顶盖套在底盒外部（或内部）
  - 通常为两部分组装
- **应用场景**: 
  1. 高档礼品盒
  2. 鞋盒
  3. 服装包装盒
  4. 精密仪器包装
- **优点**:
  1. 外观精美，适合高端产品
  2. 支撑强度好（双层壁）
  3. 开启体验好（类似取下盖子）
- **缺点**:
  1. 成本较高（两套模具/两次成型）
  2. 组装需两步（先装底盒，再套顶盖）
  3. 占用空间大（套盖部分增加高度）
- **制作成本估算**: 中高（3/5）
- **所需生产设备**: 模切机、自动粘盒机、可能需要手工组装
- **适用瓦楞纸板**: E瓦、F瓦、BE瓦；单层（用于精美印刷）、双层
- **抗压强度**: 中高（双层壁结构）
- **组装难度**: 中等
- **封闭性**: 优秀（套盖紧密）
- **市场占比**: ~6%

### 4.5 Folder Box (One-piece Folder) - 折叠纸箱
- **标准代码**: FOLDR (或 OF)
- **英文名称**: One-piece Folder / Folder Box
- **中文名称**: 折叠纸箱 / 一体式折叠盒
- **展开图**:
```
    ┌─────────────────────────────────────┐
    │    [前壁]   [底面]   [后壁]   [盖]  │
    │  ┌──────┬──────────┬──────┬────┤ │
    │  │      │          │      │    │ │
    │  └──────┴──────────┴──────┴────┘ │
    │    [左壁]    [右壁]               │
    │      │         │                  │
    │      └─────────┘                  │
    │                                     │
    └─────────────────────────────────────┘
      ← 一体式设计，折叠即成盒 →
```
- **结构特点**: 
  - 一体式设计，一张纸板折叠成型
  - 无粘合或只用少量胶带
  - 可完全折叠平放
- **应用场景**: 
  1. 披萨盒
  2. 邮寄封套（邮资盒）
  3. 服装邮寄袋
  4. 临时包装（展会、活动）
- **优点**:
  1. 可折叠平放，储运成本极低
  2. 组装极快（几秒钟）
  3. 环保（少用胶水）
- **缺点**:
  1. 强度较低（无粘合或少量粘合）
  2. 不适合重型物品
  3. 外观较简单
- **制作成本估算**: 低（1/5）
- **所需生产设备**: 模切机（准确裁切折叠线）
- **适用瓦楞纸板**: B瓦、E瓦、F瓦；单层
- **抗压强度**: 低
- **组装难度**: 非常简单
- **封闭性**: 一般（通常用胶带或舌扣）
- **市场占比**: ~4%

### 4.6 Die-cut Box - 模切纸箱
- **标准代码**: DC (Die-Cut)
- **英文名称**: Die-cut Box
- **中文名称**: 模切纸箱 / 异形纸箱
- **展开图**:
```
    ┌─────────────────────────────────────┐
    │    （根据产品设计异形展开图）        │
    │      ╱─────╲                       │
    │     │ 特殊形状 │  插舌              │
    │      ╲─────╱                       │
    │   ╱──────────────╲                 │
    │  │  锁底结构       │ 防尘翼         │
    │   ╲──────────────╱                 │
    │                                     │
    └─────────────────────────────────────┘
      ← 使用模具切割出精确形状 →
```
- **结构特点**: 
  - 使用钢模切割出精确形状
  - 可设计插舌、锁底、防尘翼等结构
  - 通常无需胶带即可自行锁合
- **应用场景**: 
  1. 电商快递盒（如京东、天猫定制盒）
  2. 电子产品包装（手机、平板）
  3. 化妆品礼盒
  4. 食品外卖包装
- **优点**:
  1. 结构灵活，可定制任意形状
  2. 无需胶带（插舌自锁）
  3. 外观精美，提升品牌形象
- **缺点**:
  1. 模具成本高（需开模）
  2. 小批量不经济
  3. 设计复杂，需专业结构工程师
- **制作成本估算**: 中高（3-4/5，含模具分摊）
- **所需生产设备**: 模切机（平压平或圆压圆）、自动粘盒机
- **适用瓦楞纸板**: 根据设计选择；常用E瓦、B瓦、BC瓦；单层、双层
- **抗压强度**: 中到高（取决于设计）
- **组装难度**: 简单到中等（插舌自锁）
- **封闭性**: 好到优秀
- **市场占比**: ~10%（增长中，电商推动）

### 4.7 Bliss Box - 布利斯纸箱
- **标准代码**: BLS
- **英文名称**: Bliss Box
- **中文名称**: 布利斯纸箱 / 折叠式托盘箱
- **展开图**:
```
    展开图（三部分组装）
    ┌─────────────────────────────────────┐
    │   [托盘底座]                        │
    │  ┌──────────────────────────┐      │
    │  │                          │      │
    │  └──────────────────────────┘      │
    │   [端壁面板]    [端壁面板]          │
    │  ┌──────────┐  ┌──────────┐       │
    │  │          │  │          │       │
    │  └──────────┘  └──────────┘       │
    │   [顶盖面板]                        │
    │  ┌──────────────────────────┐      │
    │  │                          │      │
    │  └──────────────────────────┘      │
    └─────────────────────────────────────┘
      ← 三片式组装 →
```
- **结构特点**: 
  - 三片式设计：底座、两个端壁、顶盖
  - 可折叠平放，现场组装
  - 组装后形成坚固的箱体
- **应用场景**: 
  1. 大型机械出口包装
  2. 汽车零部件包装
  3. 家具包装
  4. 军品包装（符合MIL标准）
- **优点**:
  1. 折叠后体积小小，节省回程运输成本
  2. 组装后强度极高（适合重型货物）
  3. 可根据货物尺寸调整（柔性设计）
- **缺点**:
  1. 组装较复杂（需3-5分钟）
  2. 需配件（金属扣件或螺栓）
  3. 成本较高

## 7. 技术规范 (Technical Specifications)

### 7.1 技术栈
- **前端**: 纯 HTML + CSS + JavaScript（与现有Carton-CRM一致）
- **样式**: 
  - CSS变量（主题色、字体等）
  - Flexbox + Grid 布局
  - 媒体查询（响应式）
- **交互**: 
  - 原生JavaScript（ES6+）
  - 避免使用框架（保持轻量）
- **数据存储**: 
  - 纸箱类型数据存储在独立的 `js/carton-wiki-data.js` 文件中
  - 用户收藏和笔记存储在 `localStorage`

### 7.2 文件结构
```
E:\Documents\AI\LLM\Carton-CRM\
├── index.html                  (主文件，添加侧边栏入口)
├── css\
│   ├── style.css              (全局样式)
│   └── carton-wiki.css        (纸箱Wiki模块样式)
├── js\
│   ├── app.js                 (主应用逻辑)
│   ├── carton-wiki.js         (纸箱Wiki模块逻辑)
│   └── carton-wiki-data.js    (纸箱类型数据库)
└── assets\
    └── icons\                 (图标文件)
```

### 7.3 性能要求
- **首屏加载**: < 2秒（含15种纸箱类型的所有数据）
- **搜索响应**: < 300ms（实时搜索）
- **对比加载**: < 500ms（加载对比表格）
- **动画流畅**: 60fps（过渡动画）

### 7.4 兼容性
- **桌面浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **移动浏览器**: iOS Safari 14+, Chrome Android 90+
- **屏幕适配**: 320px - 2560px 宽度
